RAG 文档问答系统

一个基于 Python、Streamlit 和 智谱 API 的轻量级文档问答应用，适合作为 AI 应用开发作品集展示。

进入在线问答系统 查看 GitHub 仓库 返回地图

项目概览

项目支持上传 txt、md、pdf 文档，自动完成文本抽取、分块、索引构建和相似片段检索。用户提问后，系统会召回相关片段，并在配置 API Key 时调用 智谱 生成回答。

即使没有配置大模型接口，也可以展示文档解析、切块、检索和参考片段，方便演示完整的 RAG 工作流。

工作流程

页面重点展示一个可解释的 RAG 链路：先把文档变成可检索片段，再把检索结果作为回答依据，而不是直接让模型凭空回答。

01 上传文档

txt / md / pdf

02 解析切块

按段落和长度切分

03 候选召回

TF-IDF n-gram

04 语义增强

改写、摘要、重排

05 生成回答

展示引用和来源

核心能力

  • 上传并解析 txt、md、pdf 文档。
  • 按段落和字符长度进行带重叠的文本切块。
  • 使用字符 n-gram 与英文词级 n-gram 的混合 TF-IDF 建立轻量检索索引。
  • 支持 智谱 语义索引增强、跨语言查询改写和候选片段重排。
  • 展示回答、引用片段、相似度、来源文件和检索过程，便于解释与调试。

问答效果示例

用户问题

这份实习记录里提到了哪些与 Python、数据处理或工程实践相关的内容？

系统回答

系统会先召回与问题最相关的片段，再基于片段总结回答，并列出来源文件、片段相似度和引用内容，方便检查回答依据。

示例引用片段：来源文件、相似度分数和原文片段会在回答下方展开，帮助判断模型回答是否真正来自上传文档。

技术组成

  • Python 负责文档解析、切块、检索和 API 调用流程。
  • Streamlit 构建可交互 Web 界面。
  • scikit-learn 的 TfidfVectorizer 用于轻量候选召回。
  • 智谱 Chat Completions API 用于语义增强、查询改写、重排和回答生成。
